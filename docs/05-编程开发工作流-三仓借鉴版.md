# 编程开发工作流（三仓借鉴版）

日期：2026-04-13

## 1. 适用场景

用于 VSCode Codex 插件中的日常开发任务：

1. 新功能开发
2. Bug 修复
3. 重构优化
4. 测试与质量回归

## 2. 流程总览

```text
需求输入
 -> 任务切割与依赖建模（tasks.json）
 -> 子代理实现（worker）
 -> 质量门验收（reviewer / qa）
 -> keep/discard 决策
 -> 状态回写与总结
```

## 3. 五阶段执行法

1. Baseline 阶段
   - 检查可用构建/测试/静态检查命令。
   - 记录当前基线结果，避免“改前不测、改后无法对比”。
2. Planning 阶段
   - 将需求拆成可验证的小任务。
   - 在 `tasks.json` 中填入依赖关系。
3. Build 阶段
   - 主代理按依赖顺序拉起 worker 子代理执行单任务。
   - 子代理只能修改授权范围文件。
4. Gate 阶段
   - reviewer 按质量门执行验证（build/type/lint/test/review）。
   - 输出 PASS/FAIL 和证据。
5. Decide 阶段
   - PASS：任务置 `done`。
   - FAIL 且未超重试：回到 `todo` 重试。
   - FAIL 且超重试：置 `blocked` 并记录原因。

## 4. 子代理消息模板

```text
执行任务: {task_id}
目标: {goal}
修改范围: {scope}
上下文文件: {context_files}
验收标准: {acceptance_criteria}
输出: 修改文件列表 + 关键变更 + 验证结果 + 风险
```

## 5. 验收输出模板

```text
[ACCEPTANCE]
task_id: {task_id}
result: PASS|FAIL
checks:
- build: PASS|FAIL
- typecheck: PASS|FAIL
- lint: PASS|FAIL
- tests: PASS|FAIL
- review: PASS|FAIL
evidence:
- {log_or_file}
reason: {failure_reason_if_any}
next_action: advance|retry|block
```

## 6. VSCode Codex 启动口令

```text
按 AGENTS.md 和 tasks.json 执行编程开发工作流：
1) 先找出 dependencies 满足的 todo 任务；
2) 每次只执行一个任务，必要时开子代理并传完整上下文；
3) 任务完成后执行质量门并输出 PASS/FAIL 与证据；
4) 根据结果做 keep/discard 决策并回写 tasks.json；
5) 循环直到全部 done 或 blocked，最后输出总结。
```
