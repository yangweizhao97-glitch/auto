# 三仓借鉴映射说明

日期：2026-04-13

## 1. 来自 `karpathy/autoresearch`

借鉴点：

1. 循环式执行思想：执行 -> 读取结果 -> 决策 -> 下一轮。
2. 明确结果记录：每次尝试都要有结构化结果和状态。
3. keep/discard 机制：有价值的改动保留，失败或无收益改动回退/重做。
4. 预算意识：每轮控制时长，失败快速止损。

落地位置：

1. `AGENTS.md` 的 `Keep or Discard Rule`。
2. `tasks.json` 的重试与失败字段。
3. `docs/05-编程开发工作流-三仓借鉴版.md` 的 Gate/Decide 阶段。

## 2. 来自 `msitarzewski/agency-agents`

借鉴点：

1. Orchestrator 角色：由主代理统一推进阶段。
2. Dev-QA 闭环：开发和验收循环，失败带反馈重试。
3. 质量门和重试上限：防止无限返工，保留可解释的阻塞状态。
4. 上下文交接要求：每个子代理都要给明确任务边界。

落地位置：

1. `AGENTS.md` 的角色分工与 `Sub-Agent Handoff Contract`。
2. `tasks.json` 的 `dependencies` + `max_retries`。
3. `docs/04-子代理与Skills验收协议.md` 的子代理模板与回写规则。

## 3. 来自 `affaan-m/everything-claude-code`

借鉴点：

1. Codex 本地配置思路（`.codex` / agent role 的实践）。
2. 质量门命令化思路（`quality-gate`、`verify`、`loop-start` 等）。
3. 多角色协作模式（explorer/reviewer/docs-researcher）。
4. 以技能为工作流入口（skills-first）。

落地位置：

1. `AGENTS.md` 的 `reviewer` / `docs_researcher` 角色。
2. `docs/05-编程开发工作流-三仓借鉴版.md` 的质量门与执行口令。
3. `docs/03-tasks-json规范.md` 与 `docs/04-子代理与Skills验收协议.md` 的 skills 验收结构。
