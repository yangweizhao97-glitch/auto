# 成熟版运行手册

日期：2026-04-13

## 1. 现在这套东西是什么

这不是纯文档模板，而是一套可执行的自动化工作流骨架：

1. `AGENTS.md`
2. `tasks.json`
3. `.codex/`
4. `prompts/`
5. `scripts/`
6. `logs/`
7. `reports/`

## 2. 核心能力

1. 自动读取 `tasks.json`
2. 自动找出依赖满足的下一个任务
3. 自动生成 main_agent/child_agent/tester_agent 提示词包
4. 自动生成每个任务的 markdown handoff packet 与结果模板
5. 自动执行质量门并产生日志与报告
6. 自动回写任务状态
7. 自动输出汇总报告

## 3. 运行顺序

### 推荐一键入口（默认）

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Start-RequestWorkflow.ps1 -Request "实现登录接口并补测试"
```

说明：

1. 会自动执行 `Status -> Validate -> (必要时)Baseline -> Auto-PlanTasks -> Next`
2. 适合每次接收新需求时直接使用
3. 后续继续走 tester 验收和主 AGENT 决策闭环

### 查看状态

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action Status
```

现在 `Status` 会按 `request_id` 分组显示每个需求批次的状态和 ready 任务。

### 跑基线质量门

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action Baseline
```

### 选择下一个任务并生成提示词

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action Next
```

### 根据需求自动拆任务

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Auto-PlanTasks.ps1 -Request "新增导出接口并补前端按钮和测试"
```

带优先级与抢占：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Auto-PlanTasks.ps1 -Request "紧急修复支付回调失败" -Priority P0 -PreemptLowerPriority
```

### 在旧任务进行中继续追加新需求

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Auto-PlanTasks.ps1 -Request "继续新增导入接口并补后台校验"
```

说明：

1. 新需求会生成新的 `request_id`
2. 新任务会追加到 `tasks.json`
3. 不会覆盖已有任务

### 校验工作流约束

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Validate-WorkflowContract.ps1
```

建议在以下时机执行：

1. 每次 `Auto-PlanTasks` 后
2. 每次批量状态回写后
3. 交付总结前

### 手动完成一个任务后回写状态

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Set-TaskStatus.ps1 -TaskId task_003 -Status done -Evidence "cmd=npm run test -- game|result=pass|log=logs\\quality-test-xxx.log|artifact=reports\\test-report-xxx.md"
```

说明：对子代理和测试代理任务，`Status=done` 时 evidence 必须包含 markdown 报告 artifact（建议 `reports/results/*.md`）。

### 验收失败并重试

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Set-TaskStatus.ps1 -TaskId task_004 -Status todo -IncrementRetry -FailureReason "Tests failed in CI" -Evidence reports\quality-gate-xxx.json
```

### 测试失败后自动回派给子 AGENT

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-TesterFeedback.ps1 -TaskId task_006 -Result FAIL -FailureReason "Backend validation failed" -RetryTaskIds task_004 -Evidence reports\tester-fail.txt
```

说明：

1. `tester_agent` 失败后可直接回派到指定子任务
2. 如果不传 `RetryTaskIds`，会默认把该请求下的实现任务都打回优化
3. 请求状态会切成 `retrying`

### 输出总结

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action Summary
```

### 生成 PR 总结

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action PrSummary
```

### 生成交付总结

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-Workflow.ps1 -Action DeliverySummary
```

## 4. 在 VSCode Codex 中怎么配合

1. 先运行 `Status` 或 `Next`。
2. 打开 `reports/` 里生成的 prompt 文件。
3. 从 prompt 中找到 `handoff_packet`，把该 markdown 文件发给子 AGENT。
4. 子 AGENT 先读取 `handoff_packet` 执行任务，不依赖历史对话上下文。
5. 子 AGENT 完成后写入 `writeback_report`（`reports/results/*.md`）并把路径回给主 AGENT。
6. 测试 AGENT 同样读取 handoff packet 并回写 markdown 测试报告给主 AGENT。
7. 主 AGENT 决定通过总结还是重新优化。
8. 如果用户中途提出新需求，主 AGENT 会把它加入当前任务池继续调度。

注意：

1. `tasks.json` 是状态中心
2. 对 `tasks.json` 的状态回写应串行执行
3. 不要同时并发运行多个会修改状态的脚本
4. 如需并行执行子任务，主 AGENT 仍需串行回写状态
5. `in_progress` 任务默认不允许 scope 冲突，除非显式传 `-AllowScopeOverlap`
6. 子任务信息传递以 markdown handoff packet 为准，不以会话上下文为准

### 主 AGENT 做最终验收决策

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-AcceptanceDecision.ps1 -TaskId task_007 -Decision accept
```

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\Invoke-AcceptanceDecision.ps1 -TaskId task_007 -Decision retry -Reason "API contract still unstable"
```

### 测试失败后自动生成优化 prompt

`Invoke-TesterFeedback.ps1` 在失败回派时会自动为被打回的子任务生成新的优化 prompt。

## 5. 适合你当前项目的原因

1. Windows 友好，全部是 PowerShell。
2. 不依赖你先搭一个 Web 平台。
3. 保留了多代理、质量门、日志、回写这些成熟工作流能力。
